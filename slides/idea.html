Tensor-Core Diagonal Kernels
for Hamiltonian Simulation on GPUs

A storage-and-layout idea that maps the banded structure of e−iHt onto tensor-core MMA — faster than cuSPARSE, with TF32 fidelity that stays effectively exact.

Physics Hamiltonians are banded — store them by diagonal, stream them branch-free.
Measured up to q = 24 qubits (16.7M dimension), vs cuSPARSE & the HM SpGEMM baseline.

01The problem

Simulating quantum dynamics means repeatedly applying the time-evolution operator

|ψ(t)⟩ = e−iHt |ψ(0)⟩ ≈ ( Σk (−iH Δt)k/k! )steps |ψ⟩
  • The inner loop is sparse matrix–vector (SpMV, apply H to a state) and sparse matrix–matrix (SpMSpM, build Hk / the propagator).
  • Dimension is 2q — at q=24 that's a 16.7M × 16.7M operator. Memory bandwidth is everything.
  • General sparse libraries (cuSPARSE CSR) treat H as an arbitrary sparse matrix and pay for it: per-nonzero column indices, gather/indirection, scattered memory.

But physics Hamiltonians are not arbitrary.

02Key observation: they're banded

  • Local Hamiltonians (Heisenberg, TFIM, Bose–Hubbard) put their nonzeros on a small number of fixed diagonals — same offsets every row.
  • So store by diagonal (DIA): one offset + one contiguous value array per diagonal.
  • No per-nonzero column index → 4 B/nnz instead of CSR's 8 B/nnz, and access is a regular, coalesced stream — no gather.
CSR (general) DIA (banded) ┌─────────────┐ ┌─────────────┐ │ x·· x········│ │╲ ╲ ╲ │ │ ·x· ·x·······│ │ ╲ ╲ ╲ │ │ ··x ··x······│ │ ╲ ╲ ╲ │ │ x·· x·· x····│ │ ╲ ╲ ╲ │ └─────────────┘ └─────────────┘ store col idx store offset per nonzero per diagonal → scattered → contiguous stream
This is the whole idea: trade generality for the banded structure that's always there in local-Hamiltonian dynamics, then build GPU kernels that stream diagonals.

04How the tensor-core kernel works

  • Each diagonal of H is a constant offset: row i pairs with state entry x[i+off].
  • Pack the diagonal values into the A operand and the shifted state slices into B, then issue a raw mma.sync.m16n8k8.tf32 — the apply y = Hx lands on the tensor cores.
  • Hand-written PTX (not the nvcuda::wmma wrapper) so we control exactly which lanes carry the band — no library overhead.
Why it's fast: the DIA layout turns scattered CSR gathers into contiguous, coalesced loads, and the MMA absorbs the multiply-accumulate — so the kernel runs at memory speed while the tensor cores hide the arithmetic.

07Accuracy: "fast" doesn't cost fidelity

  • SpMV CUDA-core path is exact fp32 (relerr 0 on representable inputs).
  • SpMSpM is bit-exact (max|C − H@H| = 0).
  • Even the TF32 evolution stays effectively exact: infidelity 1e-9 → 6e-9 over 1000 steps to q24.
Framing: over a full 1000-step trajectory at 16.7M dimension, the accumulated infidelity is ~10−9 — below any physical observable. We report it as exact.

08When does it win? One number.

num_diags / ( 2 · avg_nnz_per_row ) ≲ 2
≲ 2 — we win

Heisenberg (1.8), TFIM, Bose–Hubbard, QUBO.
Narrow, dense bands — regular streaming dominates.

≫ 2 — cuSPARSE wins

BeH (4.8), O₂ (8.4): wide bands that are 86–93% explicit zeros. DIA streams the padding; cuSPARSE skips it.

The win is regular branch-free streaming, not raw byte count — confirmed three ways (compressed-DIA, ELL, 2D tiling all lost). These kernels are latency-bound.

09Summary & what's next

The idea, distilled

  • DIA layout + tensor-core SpMV: ~2× cuSPARSE to q24, TF32 fidelity ~1e-9.
  • gather_flat SpMSpM: 1.2–2.2× over HM, 58–446× over cuSPARSE, bit-exact.
  • Operator build: 1.25–2.5× over HM, 13–470× over cuSPARSE.
  • Clean win criterion: num_diags/(2·avg_nnz) ≲ 2.

Next (beyond this basic version)

  • Symmetric / compressed store — already extends SpMSpM reach (this deck deliberately shows the basic kernel).
  • Block-DIA + batched SpMM — fill the MMA tile with multiple state vectors (block-Krylov) so the tensor cores do useful work per load.
  • Pauli-basis SpMSpM — represent H by its Pauli terms (~280× more compact), build operators algebraically to dodge the Hk fill-in wall.

Bottom line: for local-Hamiltonian dynamics, knowing the matrix is banded is worth ~2× on the hot loop — at TF32 fidelity that's indistinguishable from exact.
